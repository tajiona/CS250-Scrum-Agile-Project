# CS250-Scrum-Agile-Project
Sprint Review and Retrospective for the SNHU Travel project using Scrum-Agile software development practices.

Throughout this course, I learned how important it is to understand user needs before developing software. One of the most effective ways to do this is by creating user stories, which help translate customer requirements into clear and manageable development tasks. During the SNHU Travel project, I created user stories for features such as personalized destination recommendations, budget filters, and destination slideshows. These stories helped keep development focused on delivering value to the user while providing clear direction for developers and testers. I also learned that software development is most effective when requirements are continuously reviewed and refined based on stakeholder feedback.

My approach to developing programs has evolved through learning Agile and Scrum methodologies. Rather than viewing software development as a linear process, I now understand the value of iterative development, continuous feedback, and collaboration. In future projects, I hope to incorporate Agile practices such as Sprint Planning, Daily Scrums, retrospectives, and continuous testing to improve communication and project outcomes. This course also reinforced the importance of being a good team member. Effective software development requires communication, accountability, and a willingness to collaborate with others. Whether serving as a Scrum Master, Product Owner, Tester, or Developer, every team member contributes to the success of the project, and strong teamwork helps ensure that customer needs are met while maintaining product quality.
